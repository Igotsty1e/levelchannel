# Style notes — что было переписано и почему

## Wave 4-mid.2 (16 июня 2026) — second pass: jargon cleanup + SEO/GEO

### Главное

- **Удалена вся техническая лексика из прозы.** Из `article.md` и
  `structured.json` убраны упоминания внутренней терминологии:
  «claim → finding → evidence → source», `check_run`,
  `scripts.check_claim_links`, `Tier A/B/C/D`, `tier-c-context-only`,
  перечисление восьми стадий пайплайна, ссылки на ветку
  `data/dry-run-2026-06-16`, GitHub-репо в основном тексте,
  упоминания `fetch`, `base64-плейсхолдер`, `rejected`. Это
  публичный материал для сайта, а не отчёт о прогоне.
- **Hero meta переписан.** Было: «Период: июнь 2025 — июнь 2026 ·
  7 источников · 8 находок · 20 претензий с полной цепочкой ⇄
  источник». Стало: «Обзор · июнь 2026 · 12 минут чтения».
  Стандартный формат для читателя.
- **Методологическая секция полностью переписана** на язык
  читателя. Заголовок «Методология и инфраструктура» →
  «Как мы это проверяли». Внутри — про сбор открытых данных,
  двухступенчатый аудит, что специально исключали и почему. Без
  имён стадий, скриптов, путей файлов.
- **Секция «Чего цифры НЕ подтверждают», layer-3** теперь
  описывает иерархию источников человеческим языком:
  «первоисточники», «отраслевая аналитика», «корпоративные кейсы»,
  «отклонённые материалы» — вместо `Tier A/B/C/D`.
- **Forbes RU исключение** переформулировано: вместо «fetch вернул
  base64-плейсхолдер вместо текста; источник помечен `rejected` на
  стадии аудита» → «публичная версия страницы оказалась недоступна
  для проверки текста. Источник в исследовании не используется».

### SEO / GEO упаковка

- `seo.json` расширен под новый контракт: добавлены `slug`,
  `canonical_url`, `locale`, `published_at`, `modified_at`,
  `reading_time_minutes`, `author` (с биографией),
  `publisher` (Level Channel), `og_image_url` + `og_image_alt`,
  `about`, `mentions`, `tldr` (5 буллетов), `faq` (6 вопросов).
- TL;DR-блок (5 буллетов из `seo.tldr`) теперь поднимается над
  hero — это первый блок, который видит читатель и который
  цитируют answer-engine'ы (ChatGPT, Perplexity, Google AI
  Overviews).
- FAQ-блок (`seo.faq`) идёт в JSON-LD `FAQPage` schema → даёт
  rich snippets в поиске + цитируемые ответы в LLM-движках.
- TOC (оглавление) автогенерируется из `structured.sections`.
- Подвал статьи перестал быть техническим колофоном — теперь это
  стандартный article footer: блок с автором + датами + временем
  чтения, CTA-полоса с подпиской на дайджест, share-стрип
  (Telegram / X / LinkedIn / VK), список источников
  читательскими ярлыками («первоисточник» / «отраслевой обзор»),
  disclaimer.

## Wave 4-mid (16 июня 2026) — first stylist pass

- **Лид-абзац перепродан.** Был: «Этот отчёт фиксирует три
  ключевых тренда…». Стал: цитата-кикер «Я не буду сравнивать
  ChatGPT с собой» + сразу контраст «треть преподавателей уже
  работает с ИИ».
- **Заголовки переделаны под TL;DR-чтение.** «Рынок онлайн-
  преподавания и ИИ» → «Учитель и нейросеть». «Что НЕ
  подтверждено и где границы данных» → «Чего цифры НЕ
  подтверждают».
- **Layer-1 поднят в драматизацию.** Везде, где возможно,
  заменили «контекст-абзац» на одно большое число + одну точную
  фразу. Пример: «80 / 33» — главный сигнал года; «−7 пп» —
  главный факт по рынку.
- **«В данной статье», «следует отметить», «таким образом»** —
  удалены.

## Что НЕ менялось

- Все числа и атрибуции — без изменений.
- Структура matryoshka (3 слоя) сохранена.
- `fact-map.json` и `claims.jsonl` не тронуты (то есть цепочка
  «вывод → данные → источник» остаётся стабильной).
